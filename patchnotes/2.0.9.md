## 2.0.9

- Переработана borrower-база:
  - для ЗАЙМись реальные ФИО и ники теперь разводятся по разным полям;
  - ники копятся в `display_names`, а актуальный ник берется отдельно;
  - старые заимись-ники без ИН мигрируются из `full_name` в `display_names`.
- Нормализованы источники данных по заемщику:
  - `source` больше не должен деградировать в технический `last writer`;
  - вынесен отдельный `source_account_tag` для служебных суффиксов вроде `Y_009`;
  - контактные источники живут отдельно от общей карточки заемщика;
  - старые `finkit_claim_pdf` / `finkit_overdue_pdf_*` приводятся к human-friendly `детали займа FinKit`.
- Для FinKit исправлены сервисные агрегаты:
  - закрытые займы теперь считаются по `closed = true`, а не по `status`;
  - просрочка считается только для незакрытых займов.
- По заемщикам добавлена и начата запись derived-статуса:
  - короткий `loan_status`;
  - отдельный `loan_status_details_json`.
- Просрочки FinKit теперь сохраняют адрес/ZIP/контакты заемщика и в общую borrower-карточку, не только в `overdue_cases`.
- Поиск и карточки заемщика теперь понимают ники ЗАЙМись:
  - можно искать по nickname;
  - в карточке и доп. инфе выводятся все известные ники с акцентом на актуальном.
- Экспорт теперь содержит `current_display_name`, `display_names`, `loan_status_details_json`, `source_account_tag`.
