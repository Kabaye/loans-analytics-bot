## 2.0.10

- Поиск заемщика объединен в один вход:
  - отдельные режимы `Поиск по базе` и `Поиск по ИН` больше не нужны в новом UI;
  - теперь бот принимает одним и тем же экраном и обычный текст, и пачку из 1-10 ИН.
- Добавлен умный разбор search-input:
  - если сообщение состоит только из ИН, пробелов и разделителей — запускается batch lookup по ИН;
  - если сообщение обычное текстовое — поиск идет по локальной borrower-базе как раньше.
- Batch lookup по ИН стал тише и дешевле:
  - сначала бот ищет карточки в локальной базе;
  - в ОПИ идет только для тех ИН, по которым локальной карточки еще нет;
  - после этого новая карточка сохраняется в базу с source `search`.
- Контакты перед batch lookup по ИН по-прежнему пытаются дотягиваться из FinKit detail backfill, если телефон/email уже можно восстановить без OPI.
- Search-контур причесан:
  - в `handlers/search.py` убрана раздвоенная UI-логика режимов;
  - старые callback-id `search_db` и `search_opi_batch` оставлены как backward-compatible aliases для старых Telegram-кнопок;
  - сервисный batch-flow вынесен в отдельный `run_document_lookup_batch(...)`.
- В `answer.txt` добавлено новое подробное описание, как теперь устроен поиск и какие места кода за него отвечают.
