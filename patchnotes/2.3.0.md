## Что изменилось

- Перевели FinKit и ЗАЙМись на новый механизм fresh-уведомлений.

## Что это значит на практике

- Для FinKit бот теперь использует `modified`, а для ЗАЙМись — `updatedAt`.
- Вместо `seen_entries` для этих двух сервисов теперь работает timestamp-watermark.
- Первый scan только seed'ит watermark и не шлет стартовый flood.
- Защита от одинаковых timestamp теперь учитывает список id на максимальном timestamp.
- Это должно убрать повторные одинаковые уведомления при временном выпадении заявки из snapshot.
